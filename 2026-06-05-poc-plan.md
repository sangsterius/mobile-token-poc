# Mobile token PoC plan

## Goal

Demonstrate the core out-of-band approval loop end-to-end. Someone initiates an action on the web page → mobile app gets notified → user approves → web page updates.

**What you keep real:** the flow, the UX, push notifications, the approval screen with transaction detail, number matching.  
**What you fake:** cryptographic device binding, RASP, biometrics, real account recovery.

**Android only.** FCM setup only; no APNs/iOS provisioning. iOS is out of scope for the PoC.

---

## Three components

### 1. Backend (~100 lines, Node.js + Express, in-memory state)

```
POST /challenge/create       → { challengeId, code: "47" }
GET  /challenge/:id/status   → "pending" | "approved" | "denied"
POST /challenge/:id/approve
POST /challenge/:id/deny
POST /device/register        → stores FCM token
```

### 2. Web page (one page, vanilla JS)

```
┌─────────────────────────────────────┐
│  Mock Payment                        │
│  To:     Jan Kowalski                │
│  Amount: 1 250,00 PLN               │
│  [Confirm Transfer]                  │
└─────────────────────────────────────┘
        ↓ after click
┌─────────────────────────────────────┐
│  Waiting for approval on your phone  │
│  Match this code in the app:  [ 47 ] │
│  ⟳ polling...                       │
└─────────────────────────────────────┘
        ↓ on approval
┌─────────────────────────────────────┐
│  ✓  Transfer approved               │
└─────────────────────────────────────┘
```

### 3. Mobile app (Expo / React Native, sideloaded APK)

1. **Register** — enter user ID, send FCM token to backend
2. **Pending approval** — receive push, show amount + payee + number-match code + Approve/Deny buttons
3. **Result** — confirmation screen

No Secure Enclave, no biometrics, no RASP. Simple UUID as device token; approval POST is unauthenticated.

---

## Acceptance criteria

- Push notification arrives on Android device within 5 seconds of web button click
- Test user matches the number code correctly on first attempt without instructions
- Approval screen shows amount, payee, and number-match code without scrolling on a 6" display
- Web page transitions to "✓ Transfer approved" within 3 seconds of mobile approval tap

## NOT in scope

- Security of device binding (no Secure Enclave / Android Keystore)
- Resistance to malware (no RASP)
- PSD2 compliance (no cryptographic signing)
- Account recovery
- iOS compatibility
